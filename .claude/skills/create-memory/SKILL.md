Scan this project thoroughly and generate (or overwrite) a `CLAUDE.md` file at the project root with structured context that will help Claude work effectively in every future conversation.

## Scanning steps

### 1. Stack and dependencies
- Read `package.json`, `go.mod`, `requirements.txt`, `Gemfile`, `pyproject.toml`, `Cargo.toml`, or equivalent
- Identify framework, language, major libraries and their versions
- Identify build tool, test runner, linter/formatter

### 2. Project structure
- List top-level directories and their purpose
- Identify where components, pages, hooks, utils, types, tests, and configs live
- Note any monorepo structure (workspaces, nx, turborepo)

### 3. Conventions
- Read 3-5 existing source files to infer:
  - Naming conventions (PascalCase components, camelCase hooks, snake_case utils, etc.)
  - File organization patterns (co-located tests, barrel exports, etc.)
  - Import style (absolute paths, aliases, relative)
  - Comment and documentation style

### 4. Design system (if frontend project)
- Look for `tailwind.config.*`, theme files, CSS variables, or design tokens
- Identify color palette, typography, spacing scale
- Check for component libraries (shadcn, MUI, Radix, Ant Design, etc.)
- Read 2-3 existing components to understand patterns (className usage, variant props, etc.)

### 5. Commands
- Read `package.json` scripts, `Makefile`, or `justfile`
- Identify how to run dev server, tests, build, lint, and format

### 6. Architecture notes
- State management approach (Zustand, Redux, Context, Jotai, etc.)
- Data fetching pattern (React Query, SWR, tRPC, REST, GraphQL)
- Auth approach if visible
- Any notable patterns or constraints visible in the code
